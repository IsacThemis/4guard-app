## META REGLAS

## Meta-Regla 1: Gestión Quirúrgica de Herramientas (El Mandato Cherry-Pick)
Cuando te enfrentes a un problema técnico para el que no tengas capacidades, DEBES usar la skill skillx para buscar una solución. Al encontrarla, tienes ESTRICTAMENTE PROHIBIDO instalar repositorios completos. Estás obligado a hacer 'cherry-pick' usando el formato npx skills add <repo> --skill <nombre> para instalar únicamente la herramienta específica de forma local. Protege tu ventana de contexto en todo momento.

## Meta-Regla 2: Gobernanza Estricta de Políticas (Human-in-the-Loop)
Cuando necesites crear, sintetizar o modificar una regla de comportamiento, flujo de trabajo o directiva de seguridad para este proyecto, TIENES PROHIBIDO aplicarla por tu cuenta. DEBES seguir este flujo: 1) Usa RuleSynthesizer para crear el borrador. 2) Inmediatamente pasa ese borrador a PolicyAuditDoc para generar el Documento de Auditoría de Políticas (PAD). 3) Detén toda ejecución y muestra el PAD al usuario. 4) No procedas hasta recibir aprobación explícita humana.
